# DataSource Hub MCP Test Cases

本文档包含用于测试 DataSource Hub MCP 功能的测试用例。每个测试用例都基于现有数据源设计,模拟真实用户场景。

## 测试用例说明

- **问题类型**: 涵盖经济分析、投资建议、政策研究等多种场景
- **数据源覆盖**: 每个测试用例都关联到项目中已有的具体数据源
- **推荐理由**: 每个用例都包含了为什么推荐特定数据源的说明,便于调试和改进搜索算法

---

## Test Case 1: 中国宏观经济与货币政策关联分析

### 用户问题
```
我想分析中国近10年的经济增长与货币政策之间的关系。具体来说,我需要研究GDP增长率、货币供应量(M1/M2)、
利率变化以及固定资产投资之间的关联性,并评估宽松货币政策对实体经济的传导效果如何。
请帮我找到合适的数据源。
```


### 应该推荐的数据源 (按优先级)
1. **National Bureau of Statistics of China (国家统计局)**
   - `sources/china/national/nbs.json`
   - **推荐理由**:
     - 提供最权威的中国GDP、固定资产投资、国民经济核算等数据
     - 时间序列从1949年至今,完全覆盖近10年需求
     - 月度更新频率,数据及时性高
     - Quality Score: 4.6/5 (所有维度均为4-5星)

2. **People's Bank of China (中国人民银行)**
   - `sources/china/finance/banking/pbc.json`
   - **推荐理由**:
     - 中央银行官方数据,提供M0/M1/M2货币供应量、基准利率、存款准备金率等核心货币政策指标
     - 包含公开市场操作、政策利率等货币政策工具数据
     - 月度更新,时效性强
     - Quality Score: 4.8/5 (权威性和及时性均为5星)
     - 与问题中"货币供应量"、"利率变化"、"货币政策"等关键词高度匹配

3. **IMF Data**
   - `sources/international/economics/imf.json` (如果存在)
   - **推荐理由**: 提供国际比较视角,可用于对照分析

### 为什么这个测试用例有价值
- **多数据源整合**: 需要同时使用宏观经济数据和金融数据
- **时间序列分析**: 需要长期历史数据
- **中国特定场景**: 测试对中国数据源的识别能力
- **关键词匹配**: 测试对"货币政策"、"GDP"、"M1/M2"等专业术语的识别

## Test Case 3: 股票市场技术分析与量化投资策略

### 用户问题
```
我想开发一个股票量化交易策略,需要以下数据:
1. 美股和A股主要股票的历史价格数据(最好有20年以上历史)
2. 技术指标数据:移动平均线(SMA/EMA)、相对强弱指标(RSI)、MACD、布林带等
3. 实时或准实时的股价数据用于回测
4. 最好有API接口,方便程序化访问
5. 交易量数据和市场情绪指标

请推荐适合量化交易的数据源。
```

### 应该推荐的数据源 (按优先级)
1. **Alpha Vantage API**
   - `sources/sectors/finance_markets/alpha-vantage.json`
   - **推荐理由**:
     - 专为量化交易和开发者设计,提供完善的JSON API
     - 覆盖20万+股票代码,20+年历史数据深度
     - 提供50+种技术指标(SMA, EMA, RSI, MACD, Bollinger Bands等),完全满足需求
     - 支持多种时间分辨率:盘中(1/5/15/30/60分钟)、日线、周线、月线
     - 实时更新频率,适合回测和实盘交易
     - 提供免费层级(25次/天)和付费层级,灵活选择
     - Quality Score: 4.2/5,文档质量和及时性均为5星
     - 包含市场情绪数据(Alpha Intelligence - 新闻情绪、涨跌幅排行)

2. **CRSP (Center for Research in Security Prices)**
   - `sources/sectors/finance_markets/crsp.json` (如果存在)
   - **推荐理由**:
     - 学术研究标准数据源,历史数据质量极高
     - 适合需要长期历史数据的回测研究

3. **Bloomberg Terminal**
   - `sources/sectors/finance_markets/bloomberg-terminal.json` (如果存在)
   - **推荐理由**: 专业级金融数据终端,数据最全面但成本较高

### 为什么这个测试用例有价值
- **API需求**: 测试对"API可用性"这一特征的识别和排序
- **技术术语**: 包含大量金融专业术语(RSI, MACD, 布林带等)
- **实时性要求**: 测试对数据更新频率的理解
- **开发者场景**: 模拟程序化访问需求,测试文档质量评分

## Test Case 4: 中国外商投资趋势与产业政策分析

### 用户问题
```
我在做一个关于中国外商投资环境的研究报告,需要以下数据:
1. 近15年中国FDI(外商直接投资)流入的行业分布和国别来源
2. 中国ODI(对外直接投资)的目的地国家和投资领域
3. 外商投资企业数量变化趋势
4. 中国电子商务和零售市场的发展数据
5. 服务贸易数据,特别是金融服务、信息技术服务等高附加值服务
6. 双边经贸协定和自贸区政策信息

请帮我找到官方数据源。
```

### 应该推荐的数据源 (按优先级)
1. **Ministry of Commerce of China (中华人民共和国商务部)**
   - `sources/china/economy/trade/mofcom.json`
   - **推荐理由**:
     - 负责外商投资和对外贸易政策,提供最官方的FDI/ODI数据
     - 数据内容完全匹配用户需求:
       * FDI流入按国家、行业、地区分类
       * ODI按目的国、行业、投资方式分类
       * 外商投资企业数量统计
       * 零售销售和电子商务数据(社会消费品零售总额、网上零售额)
       * 服务贸易按类别(运输、旅游、金融服务等)分类
       * 双边经济合作和自贸区政策
     - 时间覆盖1995-2024,完全满足"近15年"需求
     - 月度更新,数据及时
     - Quality Score: 4.4/5,权威性5星
     - 提供Excel和PDF下载

2. **National Bureau of Statistics of China (国家统计局)**
   - `sources/china/national/nbs.json`
   - **推荐理由**:
     - 提供对外贸易、消费与零售的宏观数据
     - 可作为商务部数据的交叉验证来源
     - 数据颗粒度更细(省级、市级)

3. **UNCTAD (联合国贸易和发展会议)**
   - `sources/international/trade/unctad.json` (如果存在)
   - **推荐理由**: 提供国际比较视角,了解中国在全球FDI中的地位

### 为什么这个测试用例有价值
- **精准匹配**: FDI/ODI是非常具体的需求,测试专业术语识别
- **政府数据**: 测试对政府部门数据源的优先推荐
- **多维度需求**: 同时需要投资、贸易、商务数据,测试综合匹配能力
- **时间范围**: 明确的"近15年"需求,测试时间覆盖的理解

## Test Case 5: 新冠疫情对全球卫生系统的影响研究

4. 疫情对医疗资源分配和医疗系统韧性的影响研究
5. 需要能够按照主题词(MeSH)、作者、期刊、资助机构等维度检索文献

哪里可以找到这些生物医学文献?

     - 完全覆盖用户需求:
       * 包含大量COVID-19相关研究(临床试验、疫苗研发、公共卫生政策等)
       * 提供MeSH主题词索引,支持精准检索

     - Quality Score: 5.0/5 (所有6个维度均为5星)

   - `sources/academic/health/clinicaltrials-gov.json` (如果存在)

     - 专门的临床试验注册数据库
     - 包含COVID-19疫苗和治疗药物的临床试验数据

     - 包含气候变化适应和减缓相关数据
     - 农业研发投入数据

4. **AMIS (Agricultural Market Information System)**
   - `sources/sectors/agriculture_food/amis.json`
   - **推荐理由**:
     - 专注于主要粮食作物的市场信息
     - 提供粮食供需平衡和价格预测
     - G20支持的权威平台

### 为什么这个测试用例有价值
- **跨领域**: 同时涉及农业和气候两个领域,测试多领域匹配
- **粮食安全**: 全球关注的重要议题
- **数据多样性**: 需要产量、价格、气候、政策等多类型数据
- **专业数据库**: FAOSTAT是非常专业的农业数据库,测试领域专业性识别

## 找不到数据源的测试用例 (负面测试)

以下测试用例用于验证当查询主题不在数据源覆盖范围内时,MCP工具的处理能力。

### Test Case 19: 电子竞技赛事数据

**用户问题**: 我想分析近三年全球主要电竞赛事的观众数据和奖金分布,比如英雄联盟、DOTA2这些比赛

**期望结果**:
- 返回提示: "未找到匹配的数据源"
- 建议用户尝试更宽泛的搜索词,如"娱乐产业"、"数字经济"
- 如果有相关但不精确的数据源,可以推荐但标注相关度较低

**实际可能返回的数据源** (相关度低):
- National Bureau of Statistics of China - 可能包含文化娱乐产业统计,但不会有电竞细分数据
- World Bank Open Data - 可能有数字经济相关指标,但缺少电竞专项数据

**测试目标**:
- 验证工具能识别出没有高度匹配的数据源
- 测试对用户友好的错误提示
- 评估是否会错误地推荐不相关的数据源

**为何选择此用例**: 电竞是新兴产业,传统统计数据库通常不覆盖,但又不是完全不相关的领域(与文化娱乐相关)

---

### Test Case 20: 火星探测气象数据

**用户问题**: 需要火星表面的温度、气压和风速等气象观测数据,用于研究火星气候

**期望结果**:
- 明确返回: "未找到相关数据源"
- 不应推荐地球气候数据源(如NOAA)
- 可以建议用户查询NASA或其他太空机构的专业数据库

**实际可能的错误推荐** (需要避免):
- ❌ NOAA Climate Data - 虽然包含"气候"关键词,但是地球气候数据
- ❌ NASA Earth Data - 虽然是NASA,但是地球观测数据而非火星数据

**测试目标**:
- 验证工具不会因为关键词部分匹配就推荐不相关数据源
- 测试对查询意图的准确理解(火星 vs 地球)
- 评估负面结果的提示质量

**为何选择此用例**: 完全不在当前数据源覆盖范围内,测试边界场景

## 简短自然提问测试用例 (更贴近真实场景)

以下测试用例模拟真实用户的自然提问方式,问题简短且口语化。

### Test Case 9: 金价走势投资分析

**用户问题**: 调研分析一下金价的走势并给出投资建议

**期望推荐数据源**:
- **Alpha Vantage API** - 提供黄金等大宗商品的实时和历史价格数据,支持技术分析
- **World Bank Open Data** - 提供全球经济指标,用于分析影响金价的宏观因素

**推荐理由**: Alpha Vantage提供20+年商品价格历史和技术指标API,适合投资分析;World Bank数据可以分析通胀、利率等影响金价的经济因素

---

### Test Case 10: 中美贸易关系分析

**用户问题**: 帮我分析一下中美贸易战对两国进出口的影响,特别是高科技产品领域

**期望推荐数据源**:
- **China Customs (海关总署)** - 中国进出口商品详细数据,可按国家和商品类别查询
- **Ministry of Commerce of China (商务部)** - 对外贸易政策和双边贸易统计
- **U.S. Census Bureau (美国人口普查局)** - 美国官方贸易统计

**推荐理由**: 测试跨国数据源匹配和对"高科技产品"等具体领域的识别能力

---

### Test Case 11: 全球能源转型趋势

**用户问题**: 想了解一下全球可再生能源的发展趋势,哪些国家做得比较好

**期望推荐数据源**:

- **World Bank Open Data** - 各国能源消费结构和环境指标,便于跨国比较

**推荐理由**: 测试对口语化提问("想了解一下")和开放性问题("哪些国家做得好")的理解

### Test Case 12: mRNA疫苗文献综述

**推荐理由**: 测试对学术文献数据库(而非统计数据)的识别,以及对特定技术领域的理解

### Test Case 13: 中国消费市场趋势

### Test Case 15: 外资在华投资分析

## 测试用例使用说明

### 两类测试用例的区别

**详细测试用例 (Test Case 1-8)**:
- 问题描述详细,列出具体数据需求
- 适合测试精确匹配和多维度需求理解
- 模拟专业研究人员的查询方式

**简短自然测试用例 (Test Case 9-18)**:
- 问题简短口语化,贴近日常提问
- 测试自然语言理解和意图识别
- 模拟普通用户的真实使用场景

### 推荐使用方式

1. **开发阶段**: 优先使用详细测试用例,确保核心功能正确
2. **优化阶段**: 使用简短测试用例,改进自然语言理解
3. **上线前**: 两类测试用例都要通过,确保各种场景都能处理
