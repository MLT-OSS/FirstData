## 🎯 なぜFirstDataなのか？

### AI時代の事実防衛線：「情報過多」から「真実の希少性」へ

生成AIがインターネットを再構築している今日、**「情報」はかつてないほど豊富になりましたが、「真実」は希少になっています**。

ノイズ、パッチワークコンテンツ、ハルシネーション（幻覚）がデフォルトの背景となった今、**信頼できる一次情報源は、もはや単なる参考資料ではなく、インテリジェンスシステム全体の基盤**となっています。

### 私たちの使命：AI時代の信頼できる基盤の構築

このプロジェクトは、**グローバルで権威的かつ構造化された一次情報源の知識ベース**を構築することを目的としています。

私たちは、科学研究、政府の開示、法律・規制、企業提出書類・財務報告、基準・権威ある業界資料など、各分野にわたる高信頼性の情報源を体系的に発見・集約し、**散在し、標準化されておらず、再利用が困難な元のコンテンツを、追跡可能で検証可能で引用可能な「核となる事実」に変換**し、完全な証拠チェーンとバージョン履歴を保持することで、すべての結論を「原典まで」遡ることができるようにします。

### これは大規模言語モデル時代の「事実防衛線」です

✅ **モデルに対してハルシネーション防止、汚染防止の信頼できる基盤を提供**

- 1000以上の権威あるデータソースをカバーする予定（すべて世界中の政府機関、国際機関、学術機関、権威ある業界団体からのもの）
- 100% URLバリデーションでリンクの可用性を保証

✅ **深い調査研究のために計算可能でレビュー可能な証拠チェーンの閉鎖を提供**

- 完全なアクセスパスを持つ構造化メタデータシステム
- 自動証拠追跡のためのプログラマティックアクセスをサポート

✅ **AIを「二次情報の曖昧な要約」から「一次証拠に基づく厳密な推論と引用」にアップグレード**

- MCP インテリジェント Agent が複雑なクエリを理解し、権威あるデータソースを正確に推薦
- 元のデータへの完全なパス（URL、API、ダウンロード方法）を提供
- 適切な使用を保証するためのユースケースと引用例を含む

> **GenAI時代において、クリーンなデータ > 大規模モデル。**
>
> **すべての深い思考を、検証可能な事実の上に構築しましょう。**

## 📐 メタデータ構造

各データソースには、機械可読で自動証拠チェーン構築をサポートする**構造化メタデータ**が含まれています：

### 基本情報

- **ユニークID**: 小文字、ハイフン区切りの識別子
- **名前**: 英語、中国語、現地語の名前
- **組織**: 名前、タイプ、国、公式ウェブサイト
- **説明**: 詳細なバイリンガル説明

### アクセスと発見

- **プライマリURL**: メインデータポータル
- **API情報**: 可用性、ドキュメント、認証要件
- **ダウンロードオプション**: 一括ダウンロード、サポートされている形式（CSV、Excel、JSONなど）
- **アクセスレベル**: オープン、登録必要、学術、商業、制限付き

### カバレッジの詳細

- **地理的範囲**: グローバル、地域、国家、国家以下
- **国/地域**: 特定のカバレッジエリア
- **時間範囲**: 開始年、終了年、更新頻度
- **セクター**: 経済、健康、気候、エネルギーなど
- **データコンテンツ**: 特定のデータカテゴリとコンテンツの説明

### 権威識別 - AI時代の信頼性保証

**authority_levelフィールド**: 権威レベル分類

- `government` - 政府機関
- `international` - 国際機関
- `research` - 研究機関
- `market` - 市場機関
- `commercial` - 商業機関
- `other` - その他

## 🎯 FirstData MCP

**自然言語と一次権威データを接続する - 「ラストマイル」**

---

私たちは、完全なメタデータ、アクセスパス、権威識別子を持つ権威あるデータソースの構造化知識ベースを構築しました。しかし、ほとんどのユーザーにとって、本当の課題は：膨大なデータソースの中から最も適切なものを素早く見つける方法は？データソースのウェブサイトを見つけた後、複雑な公式プラットフォーム上でターゲットデータを正確に特定する方法は？これらすべてを日常のAIワークフローにシームレスに統合する方法は？

**FirstData MCP** はこの目的のために構築されました—静的なデータソース知識ベースを動的なインテリジェントナビゲーションシステムに変換し、権威あるデータを誰にでもアクセス可能にします。

---

### 📖 なぜFirstData MCPなのか？

LLM時代において、私たちは直接答えを求めることに慣れていますが、しばしば2つのコアの痛点に直面します：

❌ **痛点1: 追跡不可能なデータソース（ハルシネーションリスク）**

- AI生成の数値は古い、不正確、または完全に捏造されている可能性がある
- 元の証拠チェーンがなく、追跡・検証ができない
- 二次引用は複数の伝言ゲームを通じて元の文脈を失う

❌ **痛点2: 専門的なデータベースへの高い障壁（検索が困難）**

- 政府や国際機関のウェブサイトは複雑なナビゲーションと不明瞭な専門用語を持つ
- どのデータソースが最も権威があり、そのカバレッジは何かがわからない
- 公式ウェブサイトを見つけた後でも、ターゲットデータを特定する方法がわからない

### 💡 私たちのソリューション

**このMCPサービスは、「曖昧な意図」から「正確なデータパス」への完全な閉鎖を構築します：**

直接的に古い可能性のある数値を生成するのではなく、**「権威あるデータガイド」**として機能し、国家統計局、世界銀行、業界白書などの信頼できる一次情報源にユーザーを導き、**ステップバイステップのクリック手順を含む検索マニュアル**を提供します。

> **データを権威に戻し、検索を迷わせない。**

### ✨ コア機能

#### 1️⃣ 権威あるソースロケーター

**「どこで見つけるか？」の問題を解決**

ユーザーの自然言語の質問（例：「2023年の中国の新エネルギー車輸出」）に基づいて、最も権威ある一次データソースのウェブサイトをインテリジェントに推薦します。

**機能のハイライト:**

- 🧠 **複雑なクエリを理解**: 地理+時間+セクター+権威などの多次元ニーズを自動識別
- 🎯 **スマート推薦**: 詳細なマッチング理由付きでトップ3〜5の最も関連性の高いデータソースを推薦
- 📊 **権威保証**: すべての推薦に権威レベル（政府/国際/研究/市場など）のタグ付け
- 🔗 **すぐに使える情報**: アクセスURL、APIドキュメント、データフォーマットを直接提供
- ✅ **完全な証拠チェーン**: クエリから生データまでのすべてのステップが追跡可能
- 🌍 **グローバルカバレッジ**: データソースは世界の主要セクターをカバーし、広さと深さのバランスを取る

**マーケティングアカウントや二次引用を除外し、公式ウェブサイト、公式レポート、または専門データベースに直接到達。**

#### 2️⃣ サイトパスファインダー

**「どのように見つけるか？」の問題を解決**

複雑な構造と不明瞭な専門用語を持つ専門的なウェブサイトに対して、ホームページからターゲットデータまでの完全なナビゲーションガイダンスを含むステップバイステップの操作マニュアルを提供します。

**機能のハイライト:**

- 📖 **ステップバイステップ操作ガイド**: 各ステップの操作対象、主要なインターフェース要素、フィルタ条件設定を明確に指定
- 🎯 **マルチパス保証**: メインパス + 代替ソリューション + APIアクセスを提供、各パスは検証済み
- 🔍 **正確な位置特定**: ボタンの場所、フィルタ設定、ダウンロード手順まで具体的、一般的な説明ではない
- 💡 **障壁を下げる**: 非専門家でも迅速に特定可能、"何時間もの探索"から"3〜5分の正確な検索"へ
- 🔄 **定期更新**: ウェブサイトの変更を手動検証し、操作パスをタイムリーに更新
- 🤖 **スマートマッチング**: ユーザーのクエリ意図に基づいて最も関連性の高い操作指示を推薦

**一般的に使用される権威あるデータソースに対する詳細なウェブサイトナビゲーションガイダンスを提供し、データ位置を正確に特定。**

### 🚀 完全な使用例: スマート検索 + 操作マニュアルワンストップソリューション

**シナリオ: 投資調査のために新規上場テクノロジー企業のIPO目論見書を見つける**

**ユーザーの質問:**

> "最近上場した国内のAIテクノロジー企業を調査したい。ビジネスモデルと財務状況を分析するためにIPO目論見書を見つける必要がある"

**✅ 完全なFirstData MCPワークフロー:**

#### 📋 ステップ1: 信頼できるデータソースのスマート検索

```
推薦データソース:

1. 香港取引所（HKEX）
   - 説明: HKEXはHKEXnewsを運営し、上場企業の目論見書、年次報告書、
           IPO情報、規制提出書類の公式プラットフォームを提供
   - アクセスURL: https://www.hkexnews.hk
   - 権威レベル: market (市場機関)
   - 文書タイプ: 目論見書、年次報告書、お知らせ
   - 証拠チェーン: 公式開示文書 → 直接引用可能

2. 中国証券監督管理委員会（CSRC）
   - アクセスURL: http://www.csrc.gov.cn
   - 海外上場届出情報照会
   - 権威レベル: government (政府機関)
```

**価値ポイント:**

- ✅ **ハルシネーションを回避**: ニュースウェブサイトではなく、公式開示プラットフォームを推薦
- ✅ **権威保証**: データソースの権威レベルを明確にタグ付け
- ✅ **重要な手がかりを提供**: データ背景を理解するための財務概要情報を事前提供

#### 📖 ステップ2: ウェブサイト操作マニュアルを取得

MCPには**操作マニュアルデータベース**が組み込まれており、一般的に使用される権威あるデータソースのウェブサイトナビゲーションとダウンロード手順をカバーしています。

**HKEXnewsウェブサイト操作指示を検索:**

```
source: https://www.hkexnews.hk
operation: 新規上場企業のIPO目論見書を見つける方法は？
```

**マニュアル検索結果:**

```
ウェブサイト操作マニュアル:

メインパス（推奨）:
1. 香港証券取引所HKEXnewsホームページを訪問（https://www.hkexnews.hk）
2. メインナビゲーションバーの"NEW LISTINGS"オプションをクリック
3. 新規上場情報ページに入る
4. 対応する市場セグメント（メインボードまたはGEM）を選択
5. 新規上場企業リストでターゲット企業名または株式コードを見つける
6. 対応する目論見書ダウンロードリンク（Prospectus）をクリック
7. PDFファイルをダウンロード

代替パス:
1. 香港証券取引所HKEXnewsホームページを訪問
2. 検索機能を使用して企業名または株式コードを入力
3. 検索結果で文書タイプを"Prospectus"としてフィルタ
4. 最新日付の目論見書を選択
5. ダウンロードリンクをクリックしてPDFファイルを取得
```

**価値ポイント:**

- ✅ **すぐに使えるガイダンス**: ステップバイステップの操作指示を提供、ウェブサイト構造の手動探索不要
- ✅ **マルチパス保証**: 代替ソリューションを提供し、文書検索の成功を保証
- ✅ **障壁を下げる**: 非専門家でも公式文書を迅速に特定可能

#### 📊 ステップ3: 公式データに基づく分析

**目論見書取得後の調査分析フレームワーク:**

目論見書で開示された公式データに基づいて、体系的な分析を実施できます：

**1. ビジネスモデル分析**

- **コアビジネス**: "事業概要"セクションから企業の主要事業と収益源を理解
- **市場ポジショニング**: ターゲット市場、顧客ベース、競争優位性を分析
- **利益モデル**: 収益構造、価格戦略、マネタイゼーションパスを特定

**2. 財務状況分析**

- **収益トレンド**: 過去の財務データをレビューし、収益成長率と持続可能性を分析
- **収益性**: 粗利益率、純利益率などの主要指標を評価
- **キャッシュフロー**: 営業キャッシュフロー、資金調達目的、資本準備金を研究

**3. リスク要因評価**

- **ビジネスリスク**: 目論見書の"リスク要因"セクションにリストされている主なリスク
- **財務リスク**: 負債レベル、流動性、支払能力
- **業界リスク**: 市場競争、技術変化、規制政策の変更

**4. 投資価値判断**

- **成長可能性**: 事業計画と市場見通しに基づいて成長余地を評価
- **バリュエーションの妥当性**: PER、PSRなどの指標を業界ピアと比較
- **調達資金の使途**: IPO資金使途計画が戦略目標と一致しているか

**✅ 完全な証拠チェーンの閉鎖:**

```
ユーザーが調査ニーズを提起
  ↓
MCPスマート検索が権威あるデータソースを推薦（HKEX）
  ↓
ウェブサイト操作マニュアルを取得（ステップバイステップナビゲーションガイダンス）
  ↓
公式目論見書をダウンロード（一次権威文書）
  ↓
公式開示データに基づいて体系的分析を実施
  ↓
すべての結論を元の文書に追跡して検証可能
```

**コア価値:**

- ✅ **ハルシネーション防止**: ニュースウェブサイトやAI生成コンテンツではなく、公式HKEX開示プラットフォームを推薦
- ✅ **検証可能**: 完全な証拠チェーン、誰でもパスに従って同じ生データを取得可能
- ✅ **すぐに使えるガイダンス**: ステップバイステップの操作指示を提供、"どこで見つけるかわからない"から"3〜5分の正確な検索"へ
- ✅ **マルチパス保証**: メインパス + 代替ソリューション + 検索機能、文書ダウンロードの成功を保証

#### 設定ガイド: プラットフォームに基づいて選択

> **📝 重要な注意**
>
> **APIキーの申請（必須）**: MCPサーバーを設定する前に、[FirstData API申請](https://firstdata.deepminer.com.cn/)を訪問して無料のAPIキーを申請してください。以下の設定例のすべての`<YOUR_FIRSTDATA_API_KEY>`を実際のAPIキーに置き換えてください。

<details>
<summary><b>Claude Desktop</b></summary>

**手動JSON設定**

1. **設定ファイルを見つける**:

   - macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
   - Windows: `%APPDATA%\Claude\claude_desktop_config.json`
   - Linux: `~/.config/Claude/claude_desktop_config.json`
2. **設定を追加**:

   ```json
   {
     "mcpServers": {
       "firstdata": {
         "type": "streamable-http",
         "url": "https://firstdata.deepminer.com.cn/mcp",
         "headers": {
           "Authorization": "Bearer <YOUR_FIRSTDATA_API_KEY>"
         }
       }
     }
   }
   ```
3. **Claude Desktopを再起動**して設定を適用

</details>

<summary><b>Cline (VS Code拡張機能)</b></summary>

1. **Cline MCP設定を開く**:

   - または直接`cline_mcp_settings.json`を編集

   ```json
       "firstdata": {
         "type": "streamable-http",

           "Authorization": "Bearer <YOUR_FIRSTDATA_API_KEY>"

     }
   }
   ```

   > **ヒント**: `autoApprove`フィールドを使用すると、一般的に使用される読み取り専用ツールが毎回確認なしで自動的に実行されます。
   >
3. **設定を保存後**、ClineがMCPサーバーを自動的にロード

参照: [Cline MCP設定ドキュメント](https://docs.cline.bot/mcp/configuring-mcp-servers)
