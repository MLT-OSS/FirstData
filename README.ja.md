## 🎯 なぜFirstDataなのか？

### AI時代における事実の防衛ライン：「情報過多」から「真実の希少性」へ

生成AIがインターネットを再構築している今日、**「情報」はかつてないほど豊富になった一方で、「真実」は希少になりつつあります**。

ノイズ、寄せ集めコンテンツ、ハルシネーション（幻覚）がデフォルトの背景となる中で、**信頼できる一次情報源は単なる参考資料ではなく、インテリジェンスシステム全体の基盤となっています**。

### 私たちのミッション：AI時代の信頼できる基盤を構築する

本プロジェクトは、**グローバルで権威ある構造化された一次情報ソースのナレッジベース**を構築することを目指しています。

科学研究、政府開示、法律・規制、企業開示・財務報告、標準・権威ある業界資料など、あらゆる分野にわたる高信頼性ソースを体系的に発見・集約し、**散在する非標準的で再利用困難なオリジナルコンテンツを、追跡可能・検証可能・引用可能な「コアファクト」に変換します**。完全な証拠チェーンとバージョン履歴を保持し、すべての結論を「原典に立ち返る」ことができます。

### これは大規模モデル時代の「事実の防衛ライン」です

✅ **ハルシネーション対策・汚染対策を施した信頼できる基盤をモデルに提供**

- 1,000以上の権威あるデータソースのカバーを計画（すべて世界各地の政府機関、国際機関、学術機関、権威ある業界団体から）
- 100% URL検証により、リンクの可用性を確保

✅ **深い調査のための計算可能・検証可能な証拠チェーンの完結を提供**

- 完全なアクセスパスを備えた構造化メタデータシステム
- 自動化された証拠追跡のためのプログラマティックアクセスをサポート

✅ **AIを「漠然とした二次情報の要約」から「一次的な根拠に基づく厳密な推論と引用」へアップグレード**

- MCPインテリジェントエージェントが複雑なクエリを理解し、権威あるデータソースを正確に推奨
- 元データへの完全なパスを提供（URL、API、ダウンロード方法）
- 適切な使用を確保するためのユースケースと引用例を含む

> **GenAI時代において、クリーンデータ > 大規模モデル。**
>
> **すべての深い思考を、検証可能な事実の上に構築しましょう。**

## 📐 メタデータ構造

各データソースには、機械可読かつ自動化された証拠チェーン構築をサポートする**構造化メタデータ**が含まれています。

### コア情報

- **ユニークID**: 小文字、ハイフン区切りの識別子
- **名称**: 英語、中国語、現地語の名称
- **組織**: 名称、種類、国、公式ウェブサイト
- **説明**: 詳細なバイリンガル説明

### アクセスと発見

- **プライマリURL**: メインデータポータル
- **API情報**: 可用性、ドキュメント、認証要件
- **ダウンロードオプション**: バルクダウンロード、サポートフォーマット（CSV、Excel、JSONなど）
- **アクセスレベル**: オープン、登録必要、学術、商業、制限付き

### カバレッジ詳細

- **地理的範囲**: グローバル、地域、国家、国内
- **国/地域**: 特定のカバレッジエリア
- **時間範囲**: 開始年、終了年、更新頻度
- **セクター**: 経済、健康、気候、エネルギーなど
- **データコンテンツ**: 特定のデータカテゴリと内容の説明

### 権威識別 - AI時代の信頼性保証

**authority_levelフィールド**: 権威レベルの分類

- `government` - 政府機関
- `international` - 国際機関
- `research` - 研究機関
- `market` - 市場機関
- `commercial` - 商業機関
- `other` - その他

**完全なスキーマ**: [schemas/datasource-schema.json](schemas/datasource-schema.json) を参照

## 🎯 FirstData MCP

**自然言語と一次権威データを繋ぐ「ラストマイル」**

---

権威あるデータソースの構造化されたナレッジベースを構築しました。各データソースには完全なメタデータ、アクセスパス、権威識別子が含まれています。しかし多くのユーザーにとって、実際の課題は次のとおりです：膨大なデータソースの中から最適なものを素早く見つけるにはどうすればよいか？データソースのウェブサイトを見つけた後、複雑な公式プラットフォーム上でどのように目的のデータを正確に見つけるか？そしてこれらすべてを日常のAIワークフローにシームレスに統合するにはどうすればよいか？

**FirstData MCP**はまさにこの目的のために作られました。静的なデータソースのナレッジベースをダイナミックなインテリジェントナビゲーションシステムに変え、権威あるデータを誰もがアクセスできるようにします。

---

### 📖 なぜFirstData MCPなのか？

LLM時代には、直接答えを求めることに慣れていますが、しばしば2つのコアペインポイントに直面します：

❌ **ペインポイント1：追跡不能なデータソース（ハルシネーションリスク）**

- AIが生成した数値は古かったり、間違っていたり、完全に作り話だったりする場合がある
- 元の証拠チェーンがなく、追跡・検証ができない
- 二次引用は何重にも語り直されることで元のコンテキストを失う

❌ **ペインポイント2：専門データベースへの高いハードル（困難な検索）**

- 政府や国際機関のウェブサイトはナビゲーションが複雑で専門用語がわかりにくい
- どのデータソースが最も権威があり、何をカバーしているかわからない
- 公式ウェブサイトを見つけた後も、目的のデータの場所がわからない

### 💡 私たちのソリューション

**このMCPサービスは「漠然とした意図」から「正確なデータパス」への完全なクロージャを構築します：**

潜在的に古くなった数値を直接生成するのではなく、**「権威あるデータガイド」**として機能し、国家統計局、世界銀行、業界白書などの信頼できる一次情報源にユーザーを誘導し、**具体的なクリック手順を含む検索マニュアルを提供します**。

> **データを権威に戻し、検索で迷わないようにしましょう。**

### ✨ コア機能

#### 1️⃣ 権威あるソースロケーター

**「どこで見つける？」問題を解決**

ユーザーの自然言語による質問（例：「2023年の中国の新エネルギー車輸出」）に基づき、最も権威ある一次データソースウェブサイトをインテリジェントに推奨します。

**機能ハイライト：**

- 🧠 **複雑なクエリを理解**：地域 + 時間 + セクター + 権威などの多次元ニーズを自動識別
- 🎯 **スマート推奨**：詳細なマッチング理由を含む上位3〜5つの最も関連性の高いデータソース
- 📊 **権威の保証**：すべての推奨に権威レベルのタグ付け（政府/国際/研究/市場など）
- 🔗 **すぐに使える情報**：アクセスURL、APIドキュメント、データフォーマットを直接提供
- ✅ **完全な証拠チェーン**：クエリから元データまでのすべてのステップを追跡可能
- 🌍 **グローバルカバレッジ**：幅広さと深さのバランスを保ちながら、世界の主要セクターをカバー

**マーケティングアカウントや二次引用をフィルタリングし、公式ウェブサイト、公式レポート、または専門データベースに直接アクセス。**

**使用例：**

#### 2️⃣ サイトパスファインダー

**「どうやって見つける？」問題を解決**

複雑な構造と難解な専門用語を持つ専門ウェブサイトのために、ホームページから対象データまでの完全なナビゲーションガイダンスを含む、ステップバイステップの操作マニュアルを提供します。

**機能ハイライト：**

- 📖 **ステップバイステップ操作ガイド**：各ステップで操作対象、主要インターフェース要素、フィルター条件の設定を明確に指定
- 🎯 **マルチパスの保証**：メインパス + 代替ソリューション + APIアクセス、各パスを検証済み
- 🔍 **正確な位置特定**：ボタンの場所、フィルター設定、ダウンロード手順まで具体的、一般的な説明ではない
- 💡 **ハードルを下げる**：非専門家でもすぐに見つけられ、「何時間もの探索」から「3〜5分の正確な検索」へ
- 🔄 **定期的な更新**：ウェブサイトの変更を手動で確認し、操作パスをタイムリーに更新
- 🤖 **スマートマッチング**：ユーザーのクエリ意図に基づいて最も関連性の高い操作手順を推奨

**よく使われる権威あるデータソースのための詳細なウェブサイトナビゲーションガイダンスを提供し、データの位置を正確に特定します。**

### 🚀 完全な使用例：スマート検索 + 操作マニュアル ワンストップソリューション

**シナリオ：投資調査のために新規上場テック企業のIPO目論見書を見つける**

**ユーザーの質問：**

> 「最近上場した国内AI技術企業について調査したい。ビジネスモデルと財務状況を分析するためにIPO目論見書を見つける必要がある」

**✅ FirstData MCPの完全なワークフロー：**

#### 📋 ステップ1：信頼できるデータソースのスマート検索

```
推奨データソース：

1. 香港交易所（HKEX）
   - 説明：HKEXはHKEXnewsを運営しており、上場企業の
           目論見書、年次報告書、IPO情報、
           規制申告書の公式プラットフォームを提供
   - アクセスURL：https://www.hkexnews.hk
   - 権威レベル：market（市場機関）
   - 文書タイプ：目論見書、年次報告書、お知らせ
   - 証拠チェーン：公式開示文書 → 直接引用可能

2. 中国証券監督管理委員会（CSRC）
   - アクセスURL：http://www.csrc.gov.cn
   - 海外上場届出情報照会
   - 権威レベル：government（政府機関）
```

**バリューポイント：**

- ✅ **ハルシネーション回避**：ニュースウェブサイトではなく、公式開示プラットフォームを推奨
- ✅ **権威の保証**：データソースの権威レベルを明確にタグ付け
- ✅ **主要な手がかりを提供**：データの背景を理解するための財務サマリー情報を事前提供

#### 📖 ステップ2：ウェブサイト操作マニュアルの取得

MCPには、よく使われる権威あるデータソースのウェブサイトナビゲーションとダウンロード手順をカバーした**操作マニュアルデータベース**が内蔵されています。

**HKEXnewsウェブサイト操作手順を検索：**

```
source: https://www.hkexnews.hk
operation: 新規上場企業のIPO目論見書を見つける方法は？
```

**マニュアル検索結果：**

```
ウェブサイト操作マニュアル：

メインパス（推奨）：
1. 香港証券取引所HKEXnewsホームページにアクセス（https://www.hkexnews.hk）
2. メインナビゲーションバーの「NEW LISTINGS」オプションをクリック
3. 新規上場情報ページに入る
4. 対応する市場セグメントを選択（メインボードまたはGEM）
5. 新規上場企業リストで対象企業名または株式コードを見つける
6. 対応する目論見書のダウンロードリンク（Prospectus）をクリック
7. PDFファイルをダウンロード

代替パス：
1. 香港証券取引所HKEXnewsホームページにアクセス
2. 検索機能を使って企業名または株式コードを入力
3. 検索結果で文書タイプを「Prospectus」としてフィルタリング
4. 最新の日付の目論見書を選択
5. ダウンロードリンクをクリックしてPDFファイルを取得
```

**バリューポイント：**

- ✅ **すぐに使えるガイダンス**：ステップバイステップの操作手順を提供し、手動でウェブサイト構造を探索する必要なし
- ✅ **マルチパスの保証**：代替ソリューションを提供し、文書の取得を確実に
- ✅ **ハードルを下げる**：非専門家でも公式文書をすぐに見つけられる

#### 📊 ステップ3：公式データに基づく分析

**目論見書取得後の調査分析フレームワーク：**

目論見書に開示された公式データに基づき、体系的な分析を実施できます：

**1. ビジネスモデル分析**

- **コアビジネス**：「事業概要」セクションから会社の主要事業と収益源を把握
- **市場ポジショニング**：ターゲット市場、顧客基盤、競争優位性を分析
- **収益モデル**：収益構造、価格戦略、収益化パスを特定

**2. 財務状況分析**

- **収益トレンド**：過去の財務データをレビューし、収益成長率と持続可能性を分析
- **収益性**：粗利益率、純利益率などの主要指標を評価
- **キャッシュフロー**：営業キャッシュフロー、資金調達の目的、資本準備金を調査

**3. リスクファクター評価**

- **ビジネスリスク**：目論見書の「リスクファクター」セクションに記載された主要リスク
- **財務リスク**：負債水準、流動性、支払能力
- **業界リスク**：市場競争、技術変化、規制政策の変更

**4. 投資価値の判断**

- **成長ポテンシャル**：ビジネス計画と市場見通しに基づき成長余地を評価
- **バリュエーションの合理性**：P/E比率、P/S比率などの指標を業界同業者と比較
- **調達資金の用途**：IPO資金の使途計画が戦略目標と一致しているか

**✅ 完全な証拠チェーンのクロージャ：**

```
ユーザーが調査ニーズを提起
  ↓
MCPスマート検索が権威あるデータソースを推奨（HKEX）
  ↓
ウェブサイト操作マニュアルを取得（ステップバイステップのナビゲーションガイダンス）
  ↓
公式目論見書をダウンロード（一次権威文書）
  ↓
公式開示データに基づいて体系的な分析を実施
  ↓
すべての結論が検証のために元の文書に追跡可能
```

**コアバリュー：**

- ✅ **ハルシネーション対策**：ニュースウェブサイトやAI生成コンテンツではなく、公式HKEX開示プラットフォームを推奨
- ✅ **検証可能**：完全な証拠チェーン、誰でも同じパスをたどって同じ元データを取得可能
- ✅ **すぐに使えるガイダンス**：ステップバイステップの操作手順を提供し、「どこで見つければよいかわからない」から「3〜5分での正確な検索」へ
- ✅ **マルチパスの保証**：メインパス + 代替ソリューション + 検索機能、文書のダウンロードを確実に

<summary><b>Claude Desktop</b></summary>

**手動JSON設定**

1. **設定ファイルの場所を確認**:

   - macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
   - Windows: `%APPDATA%\Claude\claude_desktop_config.json`
   - Linux: `~/.config/Claude/claude_desktop_config.json`
2. **設定を追加**:

   ```json
   {
     "mcpServers": {
       "firstdata": {
         "type": "streamable-http",
         "url": "https://firstdata.deepminer.com.cn/mcp",
         "headers": {
           "Authorization": "Bearer <YOUR_FIRSTDATA_API_KEY>"
         }
       }
     }
   }
   ```
3. **Claude Desktopを再起動**して設定を適用
