## 质量评估维度

数据源质量从以下5个维度进行评估：

### 1. 权威性 (Authority)
评估数据源的官方性和可信度

### 2. 数据质量 (Data Quality)
评估数据的准确性、一致性和规范性

### 3. 覆盖范围 (Coverage)
评估数据的时间跨度、地理范围和主题覆盖

### 4. 可用性 (Usability)
评估数据的可访问性和易用性

### 5. 元数据完整性 (Metadata Completeness)
评估收录信息的完整程度

## 评分体系

### 评分范围

所有维度使用 **1.0 - 5.0** 评分，允许0.5分增量。

| 评分 | 等级 | 说明 |
|------|------|------|
| 5.0 💎 | 卓越 | 该领域的顶级标准 |
| 4.5 ⭐⭐⭐⭐ | 优秀 | 超过大部分同类数据源 |
| 4.0 ⭐⭐⭐ | 良好 | 满足所有基本要求 |
| 3.5 ⭐⭐ | 合格 | 基本可用，有改进空间 |
| 3.0 ⭐ | 及格 | 刚刚达到收录门槛 |
| 2.5 | 勉强 | 存在明显问题 |
| 2.0 | 较差 | 不推荐收录 |
| < 2.0 | 不合格 | 拒绝收录 |

### 最低收录标准

数据源必须满足以下最低标准才能被收录：

- ✅ 权威性 ≥ 3.0
- ✅ 数据质量 ≥ 3.0
- ✅ 可用性 ≥ 3.0
- ✅ 元数据完整性 ≥ 70%

## 完整性评估

### 字段分类

字段分为三类：

#### 必需字段 (Required) - 必须100%

基本信息：
- `id`, `name`, `name_en`, `description`
- `url`, `organization`, `organization_en`

分类信息：
- `category`, `subcategory`, `tags`

访问信息：
- `access_type`, `license`, `cost`

技术信息：
- `data_formats`, `api_available`, `update_frequency`

质量评分：
- `authority`, `data_quality`, `coverage`

状态信息：
- `status`, `verified`, `last_verified`
- `metadata_created`, `metadata_updated`

#### 推荐字段 (Recommended) - 建议≥80%

- `country`, `language`, `geographic_coverage`
- `time_coverage_start`, `time_coverage_end`
- `api_type`, `api_documentation`
- `contact_email`, `citation`

#### 可选字段 (Optional)

- `keywords`, `subjects`, `doi`
- `api_rate_limit`, `api_authentication`
- `bulk_download`, `historical_data`
- `notes`, `changelog`

### 完整性评分公式

```
完整性 = (必需字段完成度 × 50%) + (推荐字段完成度 × 35%) + (可选字段完成度 × 15%)
```

**示例**：
- 必需字段：20/20 = 100%
- 推荐字段：17/20 = 85%
- 可选字段：8/20 = 40%

```
完整性 = (100% × 50%) + (85% × 35%) + (40% × 15%)
       = 50% + 29.75% + 6%
       = 85.75%
```

### 完整性等级

| 完整性 | 等级 | 徽章 |
|--------|------|------|
| ≥ 90% | 优秀 | ⭐⭐⭐⭐⭐ |
| ≥ 80% | 良好 | ⭐⭐⭐⭐ |
| ≥ 70% | 合格 | ⭐⭐⭐ |
| ≥ 60% | 及格 | ⭐⭐ |
| < 60% | 不合格 | ❌ |

## 准确性评估

### URL有效性

所有URL必须可访问：

| 检查项 | 要求 |
|--------|------|
| HTTP状态码 | 200, 301, 302 均可接受 |
| 响应时间 | < 10秒（正常）<br>< 30秒（可接受）<br>> 30秒（警告） |
| SSL证书 | HTTPS必须有效证书 |
| 重定向 | 最多允许3次重定向 |

### 信息准确性

| 检查项 | 标准 |
|--------|------|
| 机构名称 | 与官网一致 |
| 描述内容 | 基于官方信息 |
| 许可证 | 与官方声明一致 |
| 更新频率 | 可验证 |
| 联系方式 | 有效且官方 |

### 数据验证

- ✅ 日期格式符合 ISO 8601
- ✅ 语言代码符合 ISO 639
- ✅ 国家代码符合 ISO 3166
- ✅ 许可证使用标准名称（MIT, CC-BY等）
- ✅ URL格式正确（包含协议）

## 可用性评估

### 访问难度评分

| 评分 | 访问类型 | 说明 |
|------|----------|------|
| 5.0 | 完全开放 | 无需注册，可直接访问和下载 |
| 4.5 | 简单注册 | 仅需邮箱注册，无审核 |
| 4.0 | 需要申请 | 需填写表单，快速审批（1-3天） |
| 3.5 | 有限开放 | 部分数据开放，核心数据需注册 |
| 3.0 | 严格审核 | 需提供研究计划，审批较慢 |
| 2.5 | 部分收费 | 基础功能免费，高级功能付费 |
| 2.0 | 完全付费 | 所有功能均需付费 |

### 易用性评分

| 评分 | 标准 |
|------|------|
| 5.0 | API + 多格式 + 文档完善 + 示例代码 |
| 4.5 | API + 多格式 + 文档完善 |
| 4.0 | API 或 多格式下载 + 基本文档 |
| 3.5 | 网页下载 + 常用格式 |
| 3.0 | 仅支持网页浏览或单一格式 |
| 2.5 | 格式不友好（如仅PDF扫描件） |

### 更新及时性评分

| 评分 | 更新频率 | 说明 |
|------|----------|------|
| 5.0 | 实时/每日 | 数据实时更新或每日更新 |
| 4.5 | 每周 | 每周定期更新 |
| 4.0 | 每月 | 每月定期更新 |
| 3.5 | 每季度 | 每季度更新 |
| 3.0 | 每年 | 年度更新 |
| 2.5 | 不定期 | 更新无规律 |
| 2.0 | 已停止 | 超过1年未更新 |

## 权威性评估

### 机构类型评分

| 评分 | 机构类型 | 示例 |
|------|----------|------|
| 5.0 💎 | 国际组织、国家央行、最高统计机关 | 联合国、世界银行、中国人民银行、美国普查局 |
| 4.5 ⭐ | 部委级政府机构、顶级研究机构 | 财政部、发改委、中科院、NBER |
| 4.0 | 省级政府、知名大学、大型国企 | 省统计局、清华大学、国家电网 |
| 3.5 | 地市级政府、行业协会 | 市统计局、银行业协会 |
| 3.0 | 区县级、专业机构 | 区统计局、行业研究公司 |
| 2.5 | 第三方商业机构 | 咨询公司、数据公司 |
| 2.0 | 个人或非正式组织 | 个人整理、爱好者社区 |

### 数据来源评分

| 评分 | 数据来源 |
|------|----------|
| 5.0 | 一手数据，官方直接收集 |
| 4.5 | 一手数据，委托专业机构收集 |
| 4.0 | 官方统计报告 |
| 3.5 | 汇总整理官方数据 |
| 3.0 | 混合来源（部分官方，部分估算） |
| 2.5 | 第三方估算或模型推算 |

### 引用认可度

考虑以下因素（加分项）：

- ✅ 被学术论文广泛引用 (+0.5)
- ✅ 被国际组织报告引用 (+0.5)
- ✅ 有DOI或正式引用格式 (+0.3)
- ✅ 被政府政策文件引用 (+0.3)

## 覆盖范围评估

### 时间跨度评分

| 评分 | 时间跨度 |
|------|----------|
| 5.0 | 超过50年历史数据 + 持续更新 |
| 4.5 | 20-50年历史数据 + 持续更新 |
| 4.0 | 10-20年历史数据 + 持续更新 |
| 3.5 | 5-10年历史数据 + 持续更新 |
| 3.0 | 少于5年或已停止更新 |

### 地理覆盖评分

| 评分 | 地理范围 |
|------|----------|
| 5.0 | 全球所有国家/地区 |
| 4.5 | 全球主要国家（100+） |
| 4.0 | 区域性（如欧盟、东盟全覆盖） |
| 3.5 | 国家级全覆盖 |
| 3.0 | 国家级部分覆盖或省级 |
| 2.5 | 地市级或更小范围 |

### 主题深度评分

| 评分 | 主题覆盖 |
|------|----------|
| 5.0 | 该领域最全面的数据源 |
| 4.5 | 覆盖该领域80%以上主题 |
| 4.0 | 覆盖该领域主要主题 |
| 3.5 | 覆盖该领域部分主题 |
| 3.0 | 仅覆盖单一细分主题 |

## 综合评级

### 计算公式

```
综合得分 = (权威性 × 30%) + (数据质量 × 25%) + (覆盖范围 × 20%)
         + (可用性 × 15%) + (元数据完整性 × 10%)
```

### 等级划分

| 综合得分 | 等级 | 徽章 | 说明 |
|----------|------|------|------|
| ≥ 4.5 | S级 | 💎💎💎 | 该领域的标杆数据源 |
| 4.0-4.49 | A级 | 💎💎 | 优质数据源，强烈推荐 |
| 3.5-3.99 | B级 | 💎 | 良好数据源，值得收录 |
| 3.0-3.49 | C级 | ⭐ | 合格数据源，可以收录 |
| 2.5-2.99 | D级 | ⚠️ | 勉强合格，需改进 |
| < 2.5 | F级 | ❌ | 不合格，不予收录 |

### 示例评分

**示例1：世界银行开放数据**

| 维度 | 评分 | 理由 |
|------|------|------|
| 权威性 | 5.0 | 国际组织，官方一手数据 |
| 数据质量 | 5.0 | 数据经过严格审核，标准化 |
| 覆盖范围 | 5.0 | 全球200+国家，50+年数据 |
| 可用性 | 5.0 | 完全开放，API完善，多格式 |
| 元数据完整性 | 95% | 所有字段完整 |

```
综合得分 = (5.0 × 30%) + (5.0 × 25%) + (5.0 × 20%) + (5.0 × 15%) + (4.75 × 10%)
         = 1.5 + 1.25 + 1.0 + 0.75 + 0.475
         = 4.975 ≈ 5.0
```

**等级**: S级 💎💎💎

---

**示例2：某省级统计局**

| 维度 | 评分 | 理由 |
|------|------|------|
| 权威性 | 4.0 | 省级政府官方 |
| 数据质量 | 4.0 | 官方统计，质量可靠 |
| 覆盖范围 | 3.0 | 仅覆盖本省，10年数据 |
| 可用性 | 3.5 | 网页下载，Excel格式 |
| 元数据完整性 | 75% | 推荐字段基本完整 |

```
综合得分 = (4.0 × 30%) + (4.0 × 25%) + (3.0 × 20%) + (3.5 × 15%) + (3.75 × 10%)
         = 1.2 + 1.0 + 0.6 + 0.525 + 0.375
         = 3.7
```

**等级**: B级 💎

## 审核检查清单

### 提交前自查（贡献者）

- [ ] 所有必需字段已填写
- [ ] 推荐字段完成度 ≥ 80%
- [ ] 所有URL已验证可访问
- [ ] 描述清晰详细（≥200字）
- [ ] 评分有客观依据
- [ ] JSON格式正确
- [ ] 通过Schema验证
- [ ] 通过URL验证
- [ ] 元数据完整性 ≥ 70%
- [ ] 已更新相关文档

### 代码审查检查（审核者）

#### 1. 形式检查
- [ ] 文件路径正确
- [ ] 文件命名符合规范（kebab-case）
- [ ] JSON格式正确，无语法错误
- [ ] 通过Schema验证

#### 2. 内容检查
- [ ] 描述准确，无抄袭
- [ ] URL有效且指向正确
- [ ] 机构信息准确
- [ ] 许可证信息正确
- [ ] 数据格式描述准确

#### 3. 质量检查
- [ ] 权威性评分合理
- [ ] 数据质量评分合理
- [ ] 覆盖范围评分合理
- [ ] 标签准确且具体
- [ ] 分类正确

#### 4. 完整性检查
- [ ] 必需字段100%完成
- [ ] 推荐字段 ≥ 80%
- [ ] 元数据完整性 ≥ 70%

#### 5. 一致性检查
- [ ] 与同类数据源标准一致
- [ ] 评分与描述相符
- [ ] 多个URL格式统一

### 质量改进建议

如果数据源质量不达标，审核者应提供具体的改进建议：

**示例反馈**：

```markdown
感谢您的贡献！以下是改进建议：

1. **描述不够详细**：当前135字，建议扩展到200字以上
   - 建议补充：主要数据集、覆盖指标、更新机制

2. **推荐字段缺失**：当前完成度65%
   - 缺少：`time_coverage_start`, `api_documentation`, `citation`

3. **URL需要更新**：
   - `url` 字段的链接已失效，请更新到新域名

4. **评分需要调整**：
   - `authority` 建议从3.0调整为4.0（省级政府）
   - 请在notes中说明评分依据

完成这些改进后，质量评级将从C级提升到B级。
```

## 质量持续改进

### 定期审查

每季度审查：
- 更新URL有效性
- 检查数据源是否仍在更新
- 更新元数据（如有变化）
- 调整评分（如有变化）

鼓励用户报告：
- URL失效
- 信息过时
- 评分不合理
- 发现更好的替代数据源
