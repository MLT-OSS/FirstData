# 数据源质量评估标准

本文档定义了数据源收录的质量评估标准和评分体系。

## 评分体系概述

数据源质量评估包含两个独立的评分系统：

1. **数据源质量评分**：6 个维度评估数据源本身的质量（1-5 分）
2. **元数据完整性评分**：评估 JSON 文件的字段填写完整度（百分比）

---

## 数据源质量评分（6 个维度）

所有维度使用 **1-5 整数评分**。

| 评分 | 等级 | 说明 |
|------|------|------|
| 5 💎 | 卓越 | 该领域的顶级标准 |
| 4 ⭐⭐⭐⭐ | 优秀 | 超过大部分同类数据源 |
| 3 ⭐⭐⭐ | 良好 | 满足基本要求，可以收录 |
| 2 ⭐⭐ | 合格 | 基本可用，有改进空间 |
| 1 ⭐ | 勉强 | 刚刚达到最低标准 |

**注意**：评分必须是整数（1, 2, 3, 4, 5），不使用小数。

---

### 1. authority_level（权威性）

**定义**：数据源发布机构的官方性和可信度

**对应字段**：`quality.authority_level`

**评分标准**：

| 评分 | 机构类型 | 示例 |
|------|---------|------|
| 5 💎 | 国际组织、国家级中央政府、央行 | WHO, IMF, 中国人民银行, 国家统计局 |
| 4 ⭐⭐⭐⭐ | 部委级政府机构、顶级研究机构 | 发改委, MIT, 清华大学 |
| 3 ⭐⭐⭐ | 省级政府、知名大学、行业协会 | 上海统计局, 行业协会 |
| 2 ⭐⭐ | 地市级政府、专业研究机构 | 深圳统计局, 专业智库 |
| 1 ⭐ | 区县级、第三方商业机构 | 区级政府, 商业公司 |

**最低收录标准**：≥ 3

---

### 2. methodology_transparency（方法论透明度）

**定义**：数据收集和处理方法的透明度和可验证性

**对应字段**：`quality.methodology_transparency`

**评分标准**：

| 评分 | 透明度标准 |
|------|----------|
| 5 💎 | 完整公开方法论 + 数据收集流程 + 质量控制 + 可复现 |
| 4 ⭐⭐⭐⭐ | 公开方法论 + 数据收集流程 |
| 3 ⭐⭐⭐ | 基本方法论说明 |
| 2 ⭐⭐ | 简单提及方法 |
| 1 ⭐ | 无方法论说明 |

**最低收录标准**：≥ 3

---

### 3. update_timeliness（更新及时性）

**定义**：数据更新的频率和及时性

**对应字段**：`quality.update_timeliness`

**评分标准**：

| 评分 | 更新频率 |
|------|---------|
| 5 💎 | 实时/每日更新 |
| 4 ⭐⭐⭐⭐ | 每周/每月更新 |
| 3 ⭐⭐⭐ | 每季度/每年更新 |
| 2 ⭐⭐ | 不定期但在更新 |
| 1 ⭐ | 一次性或已停止更新 |

**最低收录标准**：≥ 3

---

### 4. data_completeness（数据完整性）

**定义**：数据的覆盖范围和完整程度

**对应字段**：`quality.data_completeness`

**评分标准**：

| 评分 | 完整性标准 |
|------|----------|
| 5 💎 | 全球/全国范围 + 50年以上历史 + 该领域最全面 |
| 4 ⭐⭐⭐⭐ | 全球/全国范围 + 20年以上历史 + 覆盖主要主题 |
| 3 ⭐⭐⭐ | 区域/省级范围 + 10年以上历史 + 覆盖核心主题 |
| 2 ⭐⭐ | 部分地区 + 5年以上历史 + 部分主题 |
| 1 ⭐ | 单一地区 + 少于5年数据 + 单一主题 |

**考虑因素**：
- **地理覆盖**：global > regional > national > subnational
- **时间跨度**：(end_year - start_year)
- **主题深度**：领域数量和指标覆盖

**最低收录标准**：≥ 3

---

### 5. documentation_quality（文档质量）

**定义**：数据文档的完整性和易用性

**对应字段**：`quality.documentation_quality`

**评分标准**：

| 评分 | 文档质量标准 |
|------|------------|
| 5 💎 | 完整文档 + 示例代码 + 用户指南 + 方法论 + 数据字典 |
| 4 ⭐⭐⭐⭐ | 完整文档 + 用户指南 + 数据字典 |
| 3 ⭐⭐⭐ | 基本文档 + 数据字典 |
| 2 ⭐⭐ | 仅有基本说明文档 |
| 1 ⭐ | 无文档或文档不清晰 |

**最低收录标准**：≥ 2

---

### 6. citation_count（引用频次）

**定义**：学术和实际应用中的引用情况，反映数据源的影响力和可信度

**对应字段**：`quality.citation_count`

**评分标准**：

| 评分 | 引用情况 |
|------|---------|
| 5 💎 | 被广泛引用，学术界和实务界的标准数据源 |
| 4 ⭐⭐⭐⭐ | 经常被引用，领域内知名度高 |
| 3 ⭐⭐⭐ | 定期被引用，有一定影响力 |
| 2 ⭐⭐ | 偶尔被引用，知名度一般 |
| 1 ⭐ | 很少被引用或无引用记录 |

**评估依据**：
- 学术论文引用次数（Google Scholar, Web of Science）
- 政策文件和报告引用
- 行业报告和分析引用
- 媒体报道引用

**最低收录标准**：≥ 1（无最低要求）

---

## 最低质量收录标准

数据源必须满足以下最低标准才能被收录：

- ✅ `authority_level` (权威性) ≥ 3
- ✅ `methodology_transparency` (方法论透明度) ≥ 3
- ✅ `update_timeliness` (更新及时性) ≥ 3
- ✅ `data_completeness` (数据完整性) ≥ 3
- ✅ `documentation_quality` (文档质量) ≥ 2
- ✅ `citation_count` (引用频次) ≥ 1

---

## 元数据完整性评分

### 定义

元数据完整性评分用于评估数据源 JSON 文件的字段填写完整程度，**不是数据源本身的质量评分**。

### 字段分类

#### 必需字段（Required）- 必须 100%
- 基本信息：`id`, `name`, `organization`, `description`
- 访问信息：`access.primary_url`, `access.access_level`
- 覆盖范围：`coverage.geographic`, `coverage.temporal`, `coverage.domains`
- 质量评分：`quality.*`（6 个维度）
- 元数据：`catalog_metadata.added_date`, `status`

#### 推荐字段（Recommended）- 建议 ≥ 80%
- `data_content`, `data_characteristics`
- `licensing.license`, `usage`
- `contact`, `related_sources`

#### 可选字段（Optional）
- `metadata.standards_followed`, `notes`
- 其他扩展字段

### 完整性计算公式

```
完整性 = (必需字段完成度 × 50%) + (推荐字段完成度 × 35%) + (可选字段完成度 × 15%)
```

### 完整性等级

| 完整性 | 等级 | 徽章 |
|--------|------|------|
| ≥ 90% | 优秀 | ⭐⭐⭐⭐⭐ |
| ≥ 80% | 良好 | ⭐⭐⭐⭐ |
| ≥ 70% | 合格 | ⭐⭐⭐ |
| ≥ 60% | 及格 | ⭐⭐ |
| < 60% | 不合格 | ❌ |

### 最低收录标准

- ✅ 必需字段：100%
- ✅ 推荐字段：≥ 80%
- ✅ 总体完整性：≥ 70%

### 使用工具

```bash
# 检查单个文件的完整性
python scripts/check_completeness.py sources/china/finance/banking/pbc.json

# 输出示例：
# 必需字段: 100% (27/27)
# 推荐字段: 100% (23/23)
# 可选字段: 82% (18/22)
# 总体完成度: 97.3%
# 评级: 优秀 ⭐⭐⭐⭐⭐
```

## 评分示例

### 示例 1：世界银行（高质量数据源）

```json
{
  "id": "world-bank",
  "quality": {
    "authority_level": 5,              // 国际组织
    "methodology_transparency": 5,     // 完整公开方法论
    "update_timeliness": 5,           // 实时更新
    "data_completeness": 5,           // 全球200+国家，50+年数据
    "documentation_quality": 5,       // 完整文档+示例
    "citation_count": 5               // 广泛引用的标准数据源
  }
}
```
**元数据完整性**：95%（优秀）

---

### 示例 2：省级统计局（中等质量数据源）

```json
{
  "id": "provincial-stats",
  "quality": {
    "authority_level": 3,              // 省级政府
    "methodology_transparency": 4,     // 公开方法论
    "update_timeliness": 4,           // 每月更新
    "data_completeness": 3,           // 覆盖本省，10年数据
    "documentation_quality": 3,       // 基本文档
    "citation_count": 2               // 偶尔被引用
  }
}
```
**元数据完整性**：78%（良好）

## 注意事项

1. **质量评分必须有据**：基于实际观察，不要猜测
2. **保守评估**：不确定时给较低分，只有真正顶级数据源才给 5 分
3. **说明评分理由**：在 `notes` 字段中记录评分依据
4. **定期更新**：数据源质量可能变化，需要定期重新评估
